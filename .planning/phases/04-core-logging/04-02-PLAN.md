**period_repository.dart** — Add to PeriodRepository:

1. **`StoredPeriodWithDays`** class (in same file, above PeriodRepository): `@immutable` with `final StoredPeriod period` and `final List<StoredDayEntry> dayEntries`. Constructor `const StoredPeriodWithDays({required this.period, required this.dayEntries})`.

2. **`StoredDayEntry`** class: `@immutable` with `final int id`, `final int periodId`, `final DayEntryData data`. Constructor `const StoredDayEntry({required this.id, required this.periodId, required this.data})`.

3. **`Stream<List<StoredPeriodWithDays>> watchPeriodsWithDays()`** — Use Drift `watch()` on periods ordered by `startUtc` descending. In `asyncMap`, for each period row, query `dayEntries` where `periodId` matches, ordered by `dateUtc` ascending. Map each day entry row using `dayEntryRowToDomain` (import from mappers). Return `StoredPeriodWithDays` combining period + day entries. Import `day_entry_mapper.dart`.

4. **`Future<bool> deletePeriod(int id)`** — Inside `_db.transaction()`: first delete from `dayEntries` where `periodId.equals(id)`, then delete from `periods` where `id.equals(id)`. Return `count > 0` from the periods delete. This ensures child rows are removed before the parent, maintaining FK integrity.

5. **`Future<int> saveDayEntry(int periodId, DayEntryData data)`** — Inside `_db.transaction()`: verify the period exists (select periods where id = periodId, if empty throw StateError). Then `_db.into(_db.dayEntries).insert(dayEntryDataToInsertCompanion(periodId, data))`. Return the new row id. Import `dayEntryDataToInsertCompanion` from mapper.

6. **`Future<bool> updateDayEntry(int dayEntryId, DayEntryData data)`** — `(_db.update(_db.dayEntries)..where((t) => t.id.equals(dayEntryId))).write(dayEntryDataToUpdateCompanion(data))`. Return `updated > 0`.

7. **`Future<bool> deleteDayEntry(int dayEntryId)`** — `(_db.delete(_db.dayEntries)..where((t) => t.id.equals(dayEntryId))).go()`. Return `count > 0`.

**ptrack_data.dart** — Add to exports: `StoredPeriodWithDays`, `StoredDayEntry` from the repository file. Also export `DayEntryData` if not already re-exported (it comes from ptrack_domain which the app already imports directly, but export for convenience).

**period_repository_test.dart** — Add new test group `'day entry CRUD and watch'`:
- `'saveDayEntry creates entry linked to period'`: insert a period, saveDayEntry with flow=medium, verify via raw select on dayEntries table.
- `'saveDayEntry for non-existent period throws'`: call saveDayEntry with periodId=999, expect StateError.
- `'updateDayEntry modifies existing entry'`: insert period, save day entry, update with different flow, verify change persisted.
- `'deleteDayEntry removes only the target entry'`: insert period, save 2 day entries, delete one, verify only one remains.
- `'deletePeriod cascades to day entries'`: insert period, save 2 day entries, deletePeriod, verify both dayEntries and period are gone.
- `'watchPeriodsWithDays emits updated list after insert'`: create a period, listen to watchPeriodsWithDays, insert a day entry, verify the stream emits a list where the period includes the new day entry. Use `expectLater` with `emitsInOrder` or collect first emission after write.
