**logging_bottom_sheet.dart** — Create `apps/ptrack/lib/features/logging/logging_bottom_sheet.dart`:

1. **Top-level function `showLoggingBottomSheet`:**
```dart
Future<void> showLoggingBottomSheet(
  BuildContext context, {
  required PeriodRepository repository,
  required PeriodCalendarContext calendar,
  StoredPeriod? existingPeriod,
  StoredDayEntry? existingDayEntry,
  DateTime? initialDate,
})
```
Calls `showModalBottomSheet<void>` with `isScrollControlled: true`, `showDragHandle: true`, `useSafeArea: true`. Builder wraps `LoggingBottomSheet` in `Padding(padding: EdgeInsets.only(bottom: MediaQuery.of(context).viewInsets.bottom))`.

2. **`LoggingBottomSheet` StatefulWidget** with params: `repository`, `calendar`, `existingPeriod?`, `existingDayEntry?`, `initialDate?`.

3. **State fields:**
   - `_selectedDate` (DateTime, defaults to today or `initialDate` or `existingDayEntry.data.dateUtc`)
   - `_isNewPeriodStart` (bool, true in create mode when starting a new period)
   - `_isSettingEndDate` (bool, true when adding end date to an open period)
   - `_flowIntensity` (FlowIntensity?, pre-filled from existing day entry)
   - `_painScore` (PainScore?, pre-filled from existing)
   - `_mood` (Mood?, pre-filled from existing)
   - `_notesController` (TextEditingController, pre-filled from existing)
   - `_isSaving` (bool)
   - `_errorText` (String?, for inline validation errors)
   - `_isEditMode` (bool, derived from existingPeriod != null)

4. **Date picker section:**
   - Display current selected date with a "Change date" TextButton
   - `showDatePicker` with `firstDate: DateTime.now().subtract(Duration(days: 365 * 5))`, `lastDate: DateTime.now()` (future dates blocked per CONTEXT)
   - After picking, update `_selectedDate` and clear `_errorText`

5. **Period action section (create mode only):**
   - If no `existingPeriod`: show "Start new period on this date" as the primary action
   - If an open period exists in the repository: also offer "End [period start date] period on this date"
   - Use `Radio<bool>` or simple toggle to choose between starting new vs ending existing

6. **Day detail section** (visible in both create and edit modes, all optional):
   - **Flow intensity**: `SegmentedButton<FlowIntensity>` with 3 segments (Light, Medium, Heavy). `emptySelectionAllowed: true`, `showSelectedIcon: false`.
   - **Pain score**: `SegmentedButton<PainScore>` with 5 segments using labels. `emptySelectionAllowed: true`.
   - **Mood**: Load `MoodDisplayMode` from `MoodSettings.load()` in `initState` (cache in state field `_moodDisplayMode`). If `emoji` (default): row of 5 `ChoiceChip` widgets showing emoji (😢😟😐🙂😄). If `wordChip`: row of 5 `ChoiceChip` widgets showing text labels ("Very Bad", "Bad", "Neutral", "Good", "Very Good") per CONTEXT decision on word-chip accessibility. Single-select.
   - **Notes**: `TextFormField` with `maxLines: 3`, `decoration: InputDecoration(hintText: 'Notes (optional)', border: OutlineInputBorder())`.
   - Each section has a `Text` label above it (e.g. "Flow", "Pain", "Mood", "Notes").

7. **Hybrid validation in `_save()`:**
   - **Live inline** (checked on date change): if `_isSettingEndDate` and `_selectedDate` is before the open period's start → set `_errorText = 'End date cannot be before start date'` and return.
   - **On save**: Build `PeriodSpan` from dates. Call `repository.insertPeriod` (new) or `repository.updatePeriod` (edit). If result is `PeriodWriteRejected`, map `issues` to user-friendly error strings:
     - `OverlappingPeriod` → "This overlaps with your [start–end] period. Please adjust dates."
     - `DuplicateStartCalendarDay` → "A period already starts on this date."
     - `EndBeforeStart` → "End date cannot be before start date."
   - Display error via `_errorText` shown as red `Text` widget below the date section.
   - On success: if day details were set (any of flow/pain/mood/notes non-null), call `repository.saveDayEntry` (or `updateDayEntry` in edit mode) for `_selectedDate` with the symptom values. Then `Navigator.pop(context)` to close the bottom sheet.

8. **Save button**: `FilledButton` at the bottom. Disabled when `_isSaving`. Text: "Save" (create) or "Update" (edit).

9. The entire bottom sheet content should be in a `SingleChildScrollView` to handle keyboard resize.
