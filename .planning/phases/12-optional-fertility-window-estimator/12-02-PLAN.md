**Add to `fertility_settings.dart`:**

### FertilitySettingsTile (StatefulWidget)
Follow `MoodSettingsTile` pattern — self-contained, loads own state on init.
- Shows: ListTile with leading icon (e.g. `Icons.spa_outlined`), title from ARB `fertilitySettingsTitle`, subtitle from ARB `fertilitySettingsSubtitle`, trailing Switch.
- On toggle ON:
  1. If `disclaimerAcknowledged == false` → show disclaimer bottom sheet.
  2. After disclaimer accepted → show input collection bottom sheet.
  3. After input saved → `saveEnabled(true)`.
  4. If user dismisses either sheet without completing → toggle stays OFF.
- On toggle OFF:
  1. `saveEnabled(false)` — do NOT clear other settings (cycle length, luteal phase). Per CONTEXT: "previous settings are remembered and pre-filled from last configuration."
- Accepts optional `ValueChanged<bool>? onFertilityToggled` callback for VM notification (passed as null initially; Plan 04 wires it).

### Disclaimer bottom sheet
- `showModalBottomSheet` with `showDragHandle: true`, `useSafeArea: true`, `isScrollControlled: true` — consistent with existing bottom sheet patterns (see `_showPredictionExplanation` in `home_screen.dart`).
- Content: Title (`fertilityDisclaimerTitle`), body text (`fertilityDisclaimerBody`), "I understand" FilledButton (`fertilityDisclaimerAccept`).
- Returns `true` if accepted, `false`/null if dismissed.

### Input collection bottom sheet
- Same bottom sheet style.
- Content: Title (`fertilityInputTitle`), cycle length section, luteal phase section, Save button.
- **Cycle length:** Show auto-computed average from period history if available (requires `PeriodRepository` or pre-computed value passed in). If ≥2 completed cycles exist, display as default with hint `fertilityInputCycleLengthAutoHint`. If override exists from previous session, pre-fill that instead. Use a number input or stepper (20–45 range).
- **Luteal phase:** Slider (5–20, default 14) with label and explanation text (`fertilityInputLutealPhaseExplanation`). Show current value as "{days} days".
- **Save button:** Saves `cycleLengthOverride` and `lutealPhaseDays` via `FertilitySettings`, then pops.
- If insufficient data (< 2 cycles and no override), show `fertilityInputNotEnoughData` hint but still allow manual entry.

### Integrate into _SettingsScreen in tab_shell.dart
- Add `FertilitySettingsTile` to `_SettingsScreen`'s `ListView` — place it after `PredictionSettingsTile` and before `LockSettingsTile` (thematically adjacent to predictions). Add a `Divider()` before and after.
- Add `ValueChanged<bool>? onFertilityToggled` parameter to `_SettingsScreen` constructor, pass it to `FertilitySettingsTile`. In `_TabShellState._openSettings`, pass `onFertilityToggled: null` for now (Plan 04 wires the actual callback).
- The tile needs access to period data for auto-computing cycle length. Pass `PeriodRepository` through from `_SettingsScreen` (which gets it from `TabShell`). Add `required PeriodRepository repository` to `_SettingsScreen` constructor; pass `widget.repository` in `_openSettings`.
