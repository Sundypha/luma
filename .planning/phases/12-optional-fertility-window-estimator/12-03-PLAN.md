**FertilityDotPainter** (`calendar_painters.dart`):

Per CONTEXT decision: "small colored dot beneath the date", "soft teal/green", "different shape" (e.g. diamond or hollow circle vs solid period dot). Use Claude's discretion on exact shape — a small diamond is recommended for clear visual differentiation from both the solid-circle period dot and the hatched-circle prediction markers.

- Add `const Color kFertilityColor = Color(0xFF26A69A);` (Material teal 400 — soft teal/green per CONTEXT).
- Create `FertilityDotPainter extends CustomPainter`:
  - Paints a small diamond (rotated square) centered beneath the day number, in teal.
  - Size: approximately 6x6 logical pixels (similar to the log-marker chip).
  - Shape: diamond (45° rotated square) — distinct from the circular period dot and hatched prediction circles. This satisfies NFR-06 non-color-only requirement.
- The dot should sit in the marker strip area below the day number (same `kCalendarLogMarkerStripHeight` zone as the log-marker chip, but offset slightly if both exist).

**Update `buildCalendarDayCell`** (`calendar_painters.dart`):
- In the bottom `SizedBox(height: kCalendarLogMarkerStripHeight)` section:
  - If `data.isFertileDay && !data.hasLoggedData`: show the fertility diamond dot.
  - If `data.isFertileDay && data.hasLoggedData`: show both the log-marker chip and the fertility diamond side by side (Row with minimal spacing).
  - If `!data.isFertileDay && data.hasLoggedData`: show only the log-marker chip (existing behavior).
  - If neither: SizedBox.shrink (existing behavior).
