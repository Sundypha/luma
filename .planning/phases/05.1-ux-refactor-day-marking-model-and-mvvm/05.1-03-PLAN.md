  <action>
    **CalendarViewModel** (`ChangeNotifier`):
    - Constructor takes `PeriodRepository` + `PeriodCalendarContext`.
    - In constructor, subscribes to `repository.watchPeriodsWithDays()`. Store the `StreamSubscription`.
    - Private `_onData(List<StoredPeriodWithDays> data)` method: stores `_periodsWithDays`, calls `_recompute()`, calls `notifyListeners()`.
    - `_recompute()`: runs `PredictionCoordinator().predictNext(storedPeriods: ..., calendar: ...)` → stores `_prediction`. Runs `buildCalendarDayDataMap(periodsWithDays: ..., prediction: ..., today: DateTime.now())` → stores `_dayDataMap`.
    - Exposed read-only state:
      - `Map<DateTime, CalendarDayData> get dayDataMap`
      - `PredictionResult get prediction`
      - `List<StoredPeriodWithDays> get periodsWithDays` (for day detail sheet)
      - `DateTime focusedDay` (get/set + notifyListeners)
      - `DateTime? selectedDay` (get/set + notifyListeners)
      - `bool get showTodayButton` (derived: focusedDay not in current month)
      - `bool get showInsufficientFutureMessage` (derived: insufficient prediction + future month focused)
    - Commands:
      - `void selectDay(DateTime day)` — sets selectedDay + focusedDay, notifyListeners
      - `void changeFocusedMonth(DateTime month)` — sets focusedDay, notifyListeners
      - `void goToToday()` — resets focusedDay to now, clears selectedDay
      - `Future<void> markDay(DateTime day)` — calls `_repository.markDay(day)`
      - `Future<void> unmarkDay(DateTime day)` — calls `_repository.unmarkDay(day)`
    - `dispose()`: cancel stream subscription, call super.dispose().

    **HomeViewModel** (`ChangeNotifier`):
    - Constructor takes `PeriodRepository` + `PeriodCalendarContext`.
    - Subscribes to `repository.watchPeriodsWithDays()` in constructor.
    - `_onData`: stores data, recomputes derived state, notifyListeners.
    - Derived state:
      - `CyclePosition? cyclePosition` — from `computeCyclePosition(...)`
      - `DayEntryData? todayEntry` — searches all day entries for today's date
      - `bool isTodayMarked` — checks if any period span contains today
      - `PredictionResult prediction` — from PredictionCoordinator
      - `List<StoredPeriodWithDays> periodsWithDays`
    - Commands:
      - `Future<void> markToday()` — calls `_repository.markDay(DateTime.now())`
    - `dispose()`: cancel subscription.

    **Unit tests** (no widget rendering needed):
    - Use `MockPeriodRepository` with `mocktail`. Mock `watchPeriodsWithDays()` to return a `StreamController` you control.
    - CalendarViewModel test:
      - Starts with empty dayDataMap
      - After emitting period data → dayDataMap is populated, prediction computed
      - `selectDay` updates selectedDay + focusedDay
      - `goToToday` resets focusedDay to current month
      - dispose cancels subscription (no errors after dispose)
    - HomeViewModel test:
      - Starts with null cyclePosition, null todayEntry, isTodayMarked=false
      - After emitting data with today in a period → isTodayMarked=true, todayEntry populated
      - `markToday` calls repository.markDay with today's date
