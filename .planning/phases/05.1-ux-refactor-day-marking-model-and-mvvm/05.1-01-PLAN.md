<feature>
  <name>Day-marking span derivation — pure domain functions</name>
  <files>packages/ptrack_domain/lib/src/period/day_marking.dart, packages/ptrack_domain/test/day_marking_test.dart</files>
  <behavior>
    Six operations derived from user mark/unmark actions on calendar days:

    **computeMarkDay(existingPeriods, day):**
    - Isolated day (no adjacent period) → CreatePeriod(day)
    - Day adjacent to one period's end → ExtendPeriod(periodId, newStart, newEnd)
    - Day adjacent to one period's start → ExtendPeriod(periodId, newStart, newEnd)
    - Day bridges two periods (1-day gap) → MergePeriods(keepId, absorbId, newStart, newEnd)
    - Day already inside a period → NoOp (or error)

    **computeUnmarkDay(existingPeriods, day):**
    - Single-day period → DeletePeriod(periodId)
    - Day at start of multi-day period → ShrinkPeriod(periodId, newStart, newEnd)
    - Day at end of multi-day period → ShrinkPeriod(periodId, newStart, newEnd)
    - Day in middle of multi-day period → SplitPeriod(originalId, leftStart, leftEnd, rightStart, rightEnd)
    - Day not in any period → NoOp (or error)

    **Input types:** Use a lightweight input record with `int id` + `DateTime start` + `DateTime end` — do NOT depend on StoredPeriod from ptrack_data (domain must not import data layer). Define a small `SpanRecord` type in day_marking.dart for this.

    **Key edge cases to test:**
    - Mark day between two 1-gap periods → merge all three into one span
    - Unmark middle of 3-day period → split into two 1-day periods
    - Mark/unmark at month boundary (Dec 31 ↔ Jan 1)
    - Mark day adjacent to both start AND end of same period (single-day period next to target) → extend
    - All DateTime inputs normalized to UTC calendar midnight
  </behavior>
  <implementation>
    Create sealed class hierarchies:

    ```dart
    sealed class DayMarkOp {}
    class MarkNoOp extends DayMarkOp { ... }
    class MarkCreate extends DayMarkOp { final DateTime day; }
    class MarkExtend extends DayMarkOp { final int periodId; final DateTime newStart; final DateTime newEnd; }
    class MarkMerge extends DayMarkOp { final int keepId; final int absorbId; final DateTime newStart; final DateTime newEnd; }

    sealed class DayUnmarkOp {}
    class UnmarkNoOp extends DayUnmarkOp { ... }
    class UnmarkDelete extends DayUnmarkOp { final int periodId; }
    class UnmarkShrink extends DayUnmarkOp { final int periodId; final DateTime newStart; final DateTime newEnd; }
    class UnmarkSplit extends DayUnmarkOp { final int originalId; final DateTime leftStart; final DateTime leftEnd; final DateTime rightStart; final DateTime rightEnd; }
    ```

    Input record type (domain-layer only, no data-layer dependency):

    ```dart
    class SpanRecord {
      const SpanRecord({required this.id, required this.start, required this.end});
      final int id;
      final DateTime start; // UTC midnight
      final DateTime end;   // UTC midnight
    }
    ```

    Helper functions: `_utcDay(DateTime)`, `_nextDay(DateTime)`, `_prevDay(DateTime)`.

    `computeMarkDay`: Find adjacent periods (prev-day end or next-day start). 0 adjacent → create. 1 adjacent → extend. 2 adjacent → merge. Already contained → no-op.

    `computeUnmarkDay`: Find containing period. Not found → no-op. Single-day → delete. Edge day → shrink. Interior day → split.

    Export from `ptrack_domain.dart` barrel.
  </implementation>
</feature>
