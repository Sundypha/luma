**Create `apps/ptrack/lib/features/calendar/day_detail_sheet.dart`:**

1. `showDayDetailSheet` top-level function:
   ```dart
   Future<void> showDayDetailSheet(
     BuildContext context, {
     required DateTime selectedDay,
     required List<StoredPeriodWithDays> allData,
     required PredictionResult prediction,
     required PeriodRepository repository,
     required PeriodCalendarContext calendar,
   })
   ```
   Calls `showModalBottomSheet` with `isScrollControlled: true`, `showDragHandle: true`, `useSafeArea: true`. Builds `DayDetailSheet` widget inside.

2. `DayDetailSheet` StatefulWidget receiving the same parameters.

3. State manages:
   - `PageController _pageController` initialized to show the selected day's page
   - `DateTime _currentDay` — tracks which day is currently shown
   - The sheet computes a 3-page window: [previousDay, currentDay, nextDay]. When user swipes, the window shifts and _pageController jumps back to center.

4. **Page content per day** — `_buildDayPage(DateTime day)`:

   a. **Find the day's data**: search `allData` for a period containing this day, and for a `StoredDayEntry` matching this day's UTC calendar date.

   b. **Day with logged data (StoredDayEntry found)** — read-only view:
      - Header: formatted date (MaterialLocalizations.formatFullDate)
      - If within a period: "Period day N" label
      - Structured display of day entry fields:
        - Flow intensity: label with chip
        - Pain score: label with chip
        - Mood: emoji + label
        - Notes: full text (scrollable if long)
      - `FilledButton(onPressed: _edit, child: Text('Edit'))` at the bottom
      - `_edit()` closes this sheet, then opens `showLoggingBottomSheet(context, repository: repository, calendar: calendar, existingPeriod: period, existingDayEntry: dayEntry)`

   c. **Empty day (no logged data, not predicted)** — opens logging directly:
      - Close this sheet immediately
      - Call `showLoggingBottomSheet(context, repository: repository, calendar: calendar, initialDate: day)`
      - To avoid flicker: detect this case in `showDayDetailSheet` BEFORE opening the detail sheet. If the day is empty and not predicted, call `showLoggingBottomSheet` directly instead.

   d. **Predicted future period day** (`CalendarDayData.isPredictedPeriod == true`, no logged data):
      - Header: formatted date
      - Info card: "Period expected around this day" with `Icon(Icons.auto_awesome)` or similar
      - Prediction context: "Based on your recent cycles" (reuse PRED-04 guardrail — no medical claims)
      - `FilledButton.tonal(onPressed: _logForDay, child: Text('Log period start'))` — opens `showLoggingBottomSheet` with `initialDate: day`

5. **Swipe navigation** — `PageView` with 3 pages:
   - `previousDay = _currentDay.subtract(Duration(days: 1))`
   - `currentDay = _currentDay`
   - `nextDay = _currentDay.add(Duration(days: 1))`
   - `onPageChanged`: when user swipes to prev/next, update `_currentDay` and reset `_pageController` to center page (index 1) without animation.
   - This gives infinite-scroll illusion for adjacent days.
   - Each page built by `_buildDayPage(dayForPage)`.

6. **Delete functionality**: For days with logged data, include a `TextButton` (or trailing icon on the header) for "Delete day entry" with confirmation dialog. Reuse the same confirmation pattern from the old home_screen.dart: `showDialog` returning `bool`, call `repository.deleteDayEntry(dayEntry.id)` on confirm, then close the sheet.

**Update `apps/ptrack/lib/features/calendar/calendar_screen.dart`:**

Replace the `_openDayDetail` method:
- Import `day_detail_sheet.dart`
- Instead of calling `showLoggingBottomSheet` directly, call:
  ```dart
  void _openDayDetail(DateTime selectedDay, List<StoredPeriodWithDays> data) {
    final dayNorm = DateTime.utc(selectedDay.year, selectedDay.month, selectedDay.day);
    final dayData = _dayDataMap[dayNorm] ?? const CalendarDayData();

    // Empty non-predicted day: go straight to logging
    if (dayData.loggedPeriodState == PeriodDayState.none &&
        !dayData.isPredictedPeriod &&
        !dayData.hasLoggedData) {
      showLoggingBottomSheet(
        context,
        repository: widget.repository,
        calendar: widget.calendar,
        initialDate: selectedDay,
      );
      return;
    }

    showDayDetailSheet(
      context,
      selectedDay: selectedDay,
      allData: data,
      prediction: _cachedPrediction ?? const PredictionInsufficientHistory(
        completedCyclesAvailable: 0,
        minCompletedCyclesNeeded: 2,
      ),
      repository: widget.repository,
      calendar: widget.calendar,
    );
  }
  ```
  Store `_cachedPrediction` from the StreamBuilder's latest prediction result (the `predResult.result`) so it's available outside the builder.
