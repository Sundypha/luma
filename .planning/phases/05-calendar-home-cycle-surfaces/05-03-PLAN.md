**Create `apps/ptrack/lib/features/calendar/calendar_screen.dart`:**

1. `CalendarScreen` StatefulWidget taking `PeriodRepository repository`, `PtrackDatabase database`, `PeriodCalendarContext calendar`.

2. State fields:
   - `DateTime _focusedDay` = `DateTime.now()` (which month is shown)
   - `DateTime? _selectedDay` (currently selected day, nullable)
   - `Map&lt;DateTime, CalendarDayData&gt; _dayDataMap = const {}` (cached per-rebuild)
   - `PredictionResult? _cachedPrediction` (recomputed only on data change)

3. Build method structure:
   ```
   StreamBuilder<List<StoredPeriodWithDays>>(
     stream: widget.repository.watchPeriodsWithDays(),
     builder: (context, snapshot) {
       // Loading: show centered CircularProgressIndicator
       // Error: show error text
       // Data:
       final data = snapshot.data ?? [];
       // Compute prediction (synchronous from loaded data)
       final storedPeriods = data.map((p) => p.period).toList();
       final predResult = PredictionCoordinator()
           .predictNext(storedPeriods: storedPeriods, calendar: widget.calendar);
       // Build day data map
       _dayDataMap = buildCalendarDayDataMap(
         periodsWithDays: data,
         prediction: predResult.result,
         today: DateTime.now(),
       );
       return _buildCalendar(context, data);
     },
   )
   ```

4. `_buildCalendar(BuildContext context, List&lt;StoredPeriodWithDays&gt; data)` returns a `Column`:
   - `TableCalendar<void>(...)` in an `Expanded` widget:
     - `firstDay: DateTime.utc(2020, 1, 1)`
     - `lastDay: DateTime.utc(2030, 12, 31)`
     - `focusedDay: _focusedDay`
     - `selectedDayPredicate: (day) => _selectedDay != null && isSameDay(_selectedDay!, day)` (import `isSameDay` from table_calendar)
     - `onDaySelected: (selectedDay, focusedDay) { setState(() { _selectedDay = selectedDay; _focusedDay = focusedDay; }); _openDayDetail(selectedDay, data); }`
     - `onPageChanged: (focusedDay) { setState(() { _focusedDay = focusedDay; }); }`
     - `calendarFormat: CalendarFormat.month` (fixed month view)
     - `availableGestures: AvailableGestures.horizontalSwipe`
     - `headerStyle: HeaderStyle(formatButtonVisible: false, titleCentered: true)`
     - `calendarBuilders: CalendarBuilders(prioritizedBuilder: _dayBuilder)`
     - `startingDayOfWeek: StartingDayOfWeek.monday`
   - If `_focusedDay` is not in the current month (compare year+month with `DateTime.now()`), show a small `TextButton.icon(icon: Icon(Icons.today), label: Text('Today'), onPressed: _goToToday)` below the calendar.

5. `Widget? _dayBuilder(BuildContext context, DateTime day, DateTime focusedDay)`:
   - Normalize: `final key = DateTime.utc(day.year, day.month, day.day);`
   - Lookup: `final data = _dayDataMap[key] ?? const CalendarDayData();`
   - Return `buildCalendarDayCell(day, data)` (imported from `calendar_painters.dart`).

6. `void _goToToday()`: `setState(() { _focusedDay = DateTime.now(); _selectedDay = null; });`

7. `void _openDayDetail(DateTime selectedDay, List&lt;StoredPeriodWithDays&gt; data)`:
   - For now (before Plan 04), open the existing `showLoggingBottomSheet(context, repository: widget.repository, calendar: widget.calendar, initialDate: selectedDay)` directly. Plan 04 will replace this with the DayDetailSheet.

8. **Insufficient-prediction message**: In the StreamBuilder builder, after building the map, check if `_cachedPrediction is PredictionInsufficientHistory` and `_focusedDay` is in a future month (after today). If so, place a small `Padding` with `Text('Predictions appear after more data', style: bodySmall, color: onSurfaceVariant)` centered below the calendar grid.
