**Create `apps/ptrack/lib/features/home/cycle_position.dart`:**

1. `CyclePosition` immutable data class with fields:
   - `int dayInCycle` — days since last period started (1-based)
   - `bool isOnPeriod` — whether today falls within a period span
   - `int? periodDayNumber` — if on period, which day of the period (1-based)
   - `(DateTime, DateTime)? nextPeriodRange` — predicted next-period start range as (rangeStart, rangeEnd), null if insufficient data
   - `bool insufficientData` — true when not enough cycles for prediction (default false)

2. `CyclePosition computeCyclePosition({required List&lt;StoredPeriodWithDays&gt; periods, required PredictionResult prediction, required DateTime today})`:
   - Find the most recent period whose `startUtc` calendar day ≤ today. Use UTC-midnight normalization: `DateTime.utc(d.year, d.month, d.day)`.
   - If no periods exist: return `CyclePosition(dayInCycle: 0, isOnPeriod: false, insufficientData: true)`.
   - Calculate `dayInCycle = todayNorm.difference(lastPeriodStartNorm).inDays + 1`.
   - Determine `isOnPeriod`: true if the latest period's span `containsCalendarDayUtc(todayUtc)`.
   - If on period: `periodDayNumber = todayNorm.difference(lastPeriodStartNorm).inDays + 1`.
   - Map `prediction` to `nextPeriodRange`:
     - `PredictionInsufficientHistory` → `insufficientData: true`, `nextPeriodRange: null`
     - `PredictionRangeOnly(rangeStartUtc, rangeEndUtc)` → `(rangeStartUtc, rangeEndUtc)`
     - `PredictionPointWithRange(pointStartUtc, rangeStartUtc, rangeEndUtc)` → if range exists use `(rangeStartUtc!, rangeEndUtc!)`, else `(pointStartUtc, pointStartUtc)`
   - Import `StoredPeriodWithDays` from ptrack_data, `PredictionResult` types from ptrack_domain.
