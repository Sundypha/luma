    **Create `export_schema.dart`** in `packages/ptrack_data/lib/src/export/`:

    Define these types with `toJson()` and named `fromJson(Map<String, dynamic>)` factory constructors:

    - `const int lumaFormatVersion = 1;`

    - `class ExportOptions` — immutable, fields: `bool includePeriods`, `bool includeSymptoms` (covers flow/pain/mood), `bool includeNotes`, `String? password`. Two named constructors: `ExportOptions.everything({String? password})` (all true) and `ExportOptions.periodsOnly({String? password})` (only periods true).

    - `class LumaExportMeta` — immutable, fields: `int formatVersion`, `int schemaVersion`, `int appVersion` (as string "1.0.0+1"), `DateTime exportedAt` (UTC), `bool encrypted`, `List<String> contentTypes`. `toJson()` serializes `exported_at` via `exportedAt.toUtc().toIso8601String()`. `fromJson` validates all required keys exist, throws `FormatException` with descriptive message for missing fields.

    - `class ExportedPeriod` — immutable, fields: `int refId`, `String startUtc`, `String? endUtc`. `toJson()` omits `end_utc` key when null. `fromJson` requires `ref_id` and `start_utc`.

    - `class ExportedDayEntry` — immutable, fields: `int periodRefId`, `String dateUtc`, `int? flowIntensity`, `int? painScore`, `int? mood`, `String? notes`. `toJson()` omits null fields (snake_case keys: `period_ref_id`, `date_utc`, `flow_intensity`, `pain_score`, `mood`, `notes`). `fromJson` requires `period_ref_id` and `date_utc`.

    - `class LumaExportData` — immutable, fields: `LumaExportMeta meta`, `List<ExportedPeriod>? periods`, `List<ExportedDayEntry>? dayEntries`. Unencrypted `toJson()` returns `{"meta": ..., "data": {"periods": [...], "day_entries": [...]}}`. Encrypted form has `{"meta": ..., "payload": "<base64>"}` — but that assembly happens in ExportService, not here. `fromJson` parses `meta` and `data` sections. `fromJsonData(Map<String, dynamic> dataMap)` parses the inner `data` object.

    **Create `luma_crypto.dart`** in the same directory:

    - `class LumaCrypto` with static async methods:
      - `Future<Uint8List> encrypt(List<int> plaintext, String password)`:
        - Generate 16 random salt bytes.
        - Derive 256-bit key via `Argon2id(parallelism: 2, memory: 19456, iterations: 2, hashLength: 32)`.
        - Encrypt with `AesGcm.with256bits()`.
        - Return: `salt(16) + nonce(12) + cipherText + mac(16)` as Uint8List.
      - `Future<Uint8List> decrypt(List<int> encrypted, String password)`:
        - Extract salt (first 16), nonce (next 12), cipherText + mac (rest, mac is last 16).
        - Derive key with same KDF params.
        - Decrypt; let `SecretBoxAuthenticationError` propagate (ImportService will catch it).
    - Use `import 'package:cryptography/cryptography.dart';` only.

    **Tests:**
    - `export_schema_test.dart`: Round-trip toJson/fromJson for LumaExportMeta, ExportedPeriod, ExportedDayEntry (with and without null fields), LumaExportData. Verify null fields omitted from JSON. Verify fromJson throws FormatException on missing required fields.
    - `luma_crypto_test.dart`: Encrypt then decrypt roundtrip returns original bytes. Decrypt with wrong password throws (catch the authentication error). Encrypt produces output longer than input (salt + nonce + mac overhead).

    **Create `export_service.dart`**:

    ```dart
    typedef ProgressCallback = void Function(int current, int total);
    ```

    - `class ExportService` with constructor `ExportService(PtrackDatabase db)`.
    - `Future<ExportResult> exportData({required ExportOptions options, ProgressCallback? onProgress})`:
      1. Read all periods from DB ordered by startUtc ascending: `(select(db.periods)..orderBy([(t) => OrderingTerm.asc(t.startUtc)])).get()`.
      2. Assign sequential `refId` (1, 2, 3…) to each period. Build `List<ExportedPeriod>` mapping `startUtc` and `endUtc` to UTC ISO 8601 strings (use `.toUtc().toIso8601String()`).
      3. If `options.includePeriods` is false, skip periods and day entries entirely.
      4. Read all day entries if any symptom/notes are included: `(select(db.dayEntries)..orderBy([(t) => OrderingTerm.asc(t.dateUtc)])).get()`.
      5. Map each day entry to `ExportedDayEntry` using the period's refId (build a `Map<int, int>` from DB period.id → refId). Apply options filtering: if `!options.includeSymptoms`, null out flowIntensity/painScore/mood. If `!options.includeNotes`, null out notes. If all fields on an entry would be null (no symptoms, no notes) AND periods-only mode, skip the entry.
      6. Report progress via `onProgress` callback (current entry index / total entries).
      7. Build `LumaExportMeta` with `formatVersion: lumaFormatVersion`, `schemaVersion: ptrackSupportedSchemaVersion` (import from ptrack_database.dart), `appVersion: '1.0.0+1'`, `exportedAt: DateTime.now().toUtc()`, `encrypted: options.password != null`, `contentTypes` derived from options (e.g. `['periods', 'symptoms', 'notes']`).
      8. Build `LumaExportData(meta: meta, periods: periods, dayEntries: dayEntries)`.
      9. Serialize: `jsonEncode(exportData.toJson())` → UTF-8 bytes.
      10. If password provided, encrypt bytes via `LumaCrypto.encrypt(jsonBytes, password)`. Then build the encrypted envelope JSON: `{"meta": meta.toJson(), "payload": base64Encode(encryptedBytes)}` and encode that as UTF-8 bytes.
      11. Return `ExportResult(bytes: Uint8List, filename: 'luma-backup-YYYY-MM-DD.luma', meta: meta)` where date is from `exportedAt`.

    - `class ExportResult` — immutable, fields: `Uint8List bytes`, `String filename`, `LumaExportMeta meta`.
