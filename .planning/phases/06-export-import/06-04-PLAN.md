    **Create `import_view_model.dart`** in `apps/ptrack/lib/features/backup/`:

    Define `enum ImportStep { idle, pickingFile, passwordPrompt, previewing, chooseStrategy, importing, done, error }`.

    `class ImportViewModel extends ChangeNotifier`:

    Fields: `ImportStep _step = ImportStep.idle`, `LumaExportMeta? _meta`, `LumaExportData? _data`, `ImportPreviewResult? _preview`, `DuplicateStrategy _strategy = DuplicateStrategy.skip`, `double _progress = 0`, `String? _errorMessage`, `ImportResult? _result`, `bool _isEncrypted = false`.

    Getters for all fields. Also a getter `bool get hasData => _data != null`.

    - `Future<void> pickFile()`:
      1. Set step to `pickingFile`, notify.
      2. Call `FilePicker.platform.pickFiles(type: FileType.custom, allowedExtensions: ['luma'], withData: true)`.
      3. If user cancels (result is null), set step back to `idle`, notify, return.
      4. Get bytes from `result.files.single.bytes` (or read from path on non-web).
      5. Validate extension is `.luma`; if not, set error "Please select a .luma backup file", step to `error`, notify, return.
      6. Try `_importService.parseFileMeta(bytes)` → store meta.
      7. If `meta.encrypted`:
         - Set `_isEncrypted = true`, step to `passwordPrompt`, notify.
         - Store raw bytes for later decryption.
      8. If not encrypted:
         - Parse full data via `_importService.parseFileData(bytes)`.
         - Run preview: `ImportPreview.analyze(data, _db)`.
         - Set step to `previewing`, notify.
      9. Catch `LumaImportException` → set error message, step to `error`, notify.

    - `Future<void> submitPassword(String password)`:
      1. Try `_importService.parseFileData(storedBytes, password: password)`.
      2. If succeeds: run preview, set step to `previewing`, notify.
      3. Catch `LumaDecryptionException` → set error "Incorrect password. Please try again.", stay on `passwordPrompt` (or show inline error), notify.

    - `void selectStrategy(DuplicateStrategy strategy)`:
      Store strategy, notify.

    - `void proceedToImport()`:
      Set step to `chooseStrategy`, notify. (Called from preview step.)

    - `Future<void> applyImport(BackupService backupService, ImportService importService)`:
      1. Set step to `importing`, progress to 0, notify.
      2. Create auto-backup: `await backupService.createBackup()`.
      3. Apply: `importService.applyImport(data: _data!, strategy: _strategy, onProgress: (c, t) { _progress = c / t; notifyListeners(); })`.
      4. Store result, set step to `done`, notify.
      5. On error: set step to `error`, store message, notify.

    - `void reset()`: resets all state to idle.

    Constructor takes `ImportService importService`, `BackupService backupService`, and `PtrackDatabase db` (for preview).

    **Create `import_screen.dart`**:

    `class ImportScreen extends StatefulWidget` — takes `ImportService importService`, `BackupService backupService`, `PtrackDatabase db`.

    State creates `ImportViewModel` in initState (passing services), disposes in dispose. Calls `_vm.pickFile()` in `initState` (or after first frame via `WidgetsBinding.instance.addPostFrameCallback`) so file picker opens immediately.

    Build uses `ListenableBuilder(listenable: _vm, builder: ...)` switching on `_vm.step`:

    - **idle / pickingFile**: `Center(child: CircularProgressIndicator())` with "Selecting file..." text.

    - **passwordPrompt**:
      - Icon lock + text: "This backup is password-protected".
      - File info: date exported, content types from meta.
      - `TextField` for password (obscured).
      - "Decrypt" button → `_vm.submitPassword(password)`.
      - Error text shown inline if wrong password was tried.
      - "Cancel" → `Navigator.pop()`.

    - **previewing**:
      - Summary card: "Found {totalPeriods} periods and {totalEntries} day entries".
      - If duplicates > 0: "⚠ {duplicateEntries} entries match existing dates".
      - If duplicates == 0: "✓ No duplicates found — all entries are new".
      - "Continue" button → `_vm.proceedToImport()` (goes to strategy selection).
      - If no duplicates, skip strategy and go straight to import (with strategy=skip, irrelevant).

    - **chooseStrategy**:
      - Title: "How should duplicates be handled?"
      - Two `RadioListTile<DuplicateStrategy>`:
        - `skip`: "Keep existing" — subtitle "Entries already on your device stay unchanged. Only new dates are imported."
        - `replace`: "Use imported" — subtitle "Entries from the backup file replace your current data for matching dates."
      - Explanatory text: "Duplicate = same calendar date. {duplicateEntries} entries affected."
      - "Import" button → `_vm.applyImport(backupService, importService)`.
      - "Cancel" → `Navigator.pop()`.

    - **importing**:
      - `LinearProgressIndicator(value: _vm.progress)`.
      - "Creating safety backup..." then "Importing entries..." status text.
      - No back button.

    - **done**:
      - Success icon (check circle).
      - Summary: "{periodsCreated} periods imported, {entriesCreated} new entries, {entriesSkipped} skipped, {entriesReplaced} replaced".
      - "Done" button → `Navigator.pop()`.

    - **error**:
      - Error icon + `_vm.errorMessage`.
      - "Try Again" button → `_vm.reset()` then `_vm.pickFile()`.
      - "Close" button → `Navigator.pop()`.

    AppBar title: "Import Backup".
