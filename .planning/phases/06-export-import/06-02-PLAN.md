  <action>
    **Create `import_service.dart`**:

    Define typed error classes:
    - `class LumaImportException implements Exception` — base with `String message`.
    - `class LumaInvalidFileException extends LumaImportException` — not valid JSON or not a .luma structure.
    - `class LumaVersionException extends LumaImportException` — format_version or schema_version mismatch; include `int fileVersion` and `int supportedVersion`.
    - `class LumaDecryptionException extends LumaImportException` — wrong password or corrupted encrypted payload.

    Define enums and result types:
    - `enum DuplicateStrategy { skip, replace }`
    - `class ImportResult` — immutable: `int periodsCreated`, `int entriesCreated`, `int entriesSkipped`, `int entriesReplaced`.

    `class ImportService` with constructor `ImportService(PtrackDatabase db)`:

    - `LumaExportMeta parseFileMeta(Uint8List bytes)`:
      1. Try `utf8.decode(bytes)` → `jsonDecode` → `Map<String, dynamic>`.
      2. If fails, throw `LumaInvalidFileException('File is not a valid Luma backup')`.
      3. Extract `meta` key → `LumaExportMeta.fromJson(meta)`.
      4. Validate `meta.formatVersion == lumaFormatVersion`; if not, throw `LumaVersionException` with "created by newer version" message.
      5. Validate `meta.schemaVersion <= ptrackSupportedSchemaVersion`; if not, throw `LumaVersionException` with "please update the app" message.
      6. Return parsed meta.

    - `Future<LumaExportData> parseFileData(Uint8List bytes, {String? password})`:
      1. Parse meta via `parseFileMeta(bytes)`.
      2. If `meta.encrypted == true`:
         - If `password == null`, throw `LumaDecryptionException('This file is password-protected')`.
         - Extract `payload` (base64 string) from root JSON.
         - Decode base64 to encrypted bytes.
         - Try `LumaCrypto.decrypt(encryptedBytes, password)`.
         - Catch the authentication error from cryptography package → throw `LumaDecryptionException('Incorrect password or corrupted file')`.
         - Decode decrypted bytes as UTF-8 → jsonDecode → parse data section.
      3. If not encrypted: parse `data` section directly from root JSON.
      4. Build and return `LumaExportData(meta: meta, periods: ..., dayEntries: ...)`.

    **Create `import_preview.dart`**:

    `class ImportPreviewResult` — immutable: `int newPeriods`, `int existingPeriodOverlaps`, `int newEntries`, `int duplicateEntries`, `int totalPeriods`, `int totalEntries`.

    `class ImportPreview`:
    - `static Future<ImportPreviewResult> analyze(LumaExportData data, PtrackDatabase db)`:
      1. Load all existing day entries from DB: `(select(db.dayEntries)).get()`.
      2. Build `Set<DateTime>` of existing entry dates (normalized to UTC calendar midnight).
      3. Count imported day entries: for each, check if date exists in the set → increment `newEntries` or `duplicateEntries`.
      4. Load all existing periods from DB.
      5. For each imported period, check if any existing period overlaps the date range → count `existingPeriodOverlaps` (informational, not blocking).
      6. Return `ImportPreviewResult` with all counts.

    **Tests** (`import_service_test.dart`):
    - `parseFileMeta`: valid .luma bytes → returns correct meta.
    - `parseFileMeta`: garbage bytes → throws `LumaInvalidFileException`.
    - `parseFileMeta`: valid JSON but no `meta` key → throws `LumaInvalidFileException`.
    - `parseFileMeta`: format_version=99 → throws `LumaVersionException` with descriptive message.
    - `parseFileMeta`: schema_version > current → throws `LumaVersionException`.
    - `parseFileData`: unencrypted valid file → returns full LumaExportData.
    - `parseFileData`: encrypted file without password → throws `LumaDecryptionException`.
    - `parseFileData`: encrypted file with wrong password → throws `LumaDecryptionException`.
    - `parseFileData`: encrypted file with correct password → returns decrypted data.
    - `ImportPreview.analyze`: 3 imported entries, 1 matching existing date → newEntries=2, duplicateEntries=1.

    For encryption tests, use ExportService (from Plan 01) to create a real encrypted export, then parse it — validates the full encrypt→decrypt chain.

    **Create `backup_service.dart`**:

    `class BackupService` with constructor `BackupService(PtrackDatabase db)`:

    - `Future<File> createBackup()`:
      1. Get app support dir via `getApplicationSupportDirectory()`.
      2. Create `luma_backups/` subdirectory if not exists.
      3. Export current data using `ExportService(db).exportData(options: ExportOptions.everything())` (no password for auto-backups).
      4. Write bytes to `luma_backups/auto-backup-YYYY-MM-DD-HHmmss.luma`.
      5. Call `_pruneBackups(keepCount: 3)` after write.
      6. Return the created File.

    - `Future<List<BackupInfo>> listBackups()`:
      1. List files in `luma_backups/` matching `auto-backup-*.luma` pattern.
      2. Return sorted by modification time descending (newest first).
      3. `BackupInfo` — immutable: `String path`, `DateTime createdAt`, `int sizeBytes`.

    - `Future<void> _pruneBackups({int keepCount = 3})`:
      1. List all auto-backup files sorted by modification time descending.
      2. Delete files beyond `keepCount`.

    **Add `applyImport` to `ImportService`** (modify `import_service.dart`):

    - `Future<ImportResult> applyImport({required LumaExportData data, required DuplicateStrategy strategy, ProgressCallback? onProgress})`:
      1. Run inside `_db.transaction(() async { ... })` for atomicity.
      2. Build a `Map<int, int>` from imported `refId → new DB periodId`.
      3. For each imported period:
         - Insert into `periods` table: `into(db.periods).insert(PeriodsCompanion.insert(startUtc: parsedStartUtc, endUtc: Value(parsedEndUtc)))`.
         - Capture the returned row ID. Map `refId → newId`.
         - Increment `periodsCreated`.
      4. For each imported day entry:
         - Look up the new periodId from the refId map.
         - Normalize dateUtc to UTC calendar midnight.
         - Check for existing entry on same date: `(select(db.dayEntries)..where((t) => t.dateUtc.equals(dateUtc))).getSingleOrNull()`.
         - If exists and strategy == `skip`: increment `entriesSkipped`, continue.
         - If exists and strategy == `replace`: update the existing row with imported values, increment `entriesReplaced`.
         - If not exists: insert new row with the new periodId, increment `entriesCreated`.
      5. Report progress via `onProgress(currentIndex, totalEntries)`.
      6. Return `ImportResult`.

    **Update `ptrack_data.dart` barrel** — add exports for:
    - `import_service.dart`: ImportService, ImportResult, DuplicateStrategy, LumaImportException, LumaInvalidFileException, LumaVersionException, LumaDecryptionException
    - `import_preview.dart`: ImportPreview, ImportPreviewResult
    - `backup_service.dart`: BackupService, BackupInfo

    **Tests** (`backup_service_test.dart`):
    - Use an in-memory Drift database (not mock) for integration-style tests.
    - Test: `applyImport` with empty DB + 2 periods + 3 entries → periodsCreated=2, entriesCreated=3, no duplicates.
    - Test: `applyImport` with strategy=skip, 1 existing entry on same date → entriesSkipped=1.
    - Test: `applyImport` with strategy=replace, 1 existing entry on same date → entriesReplaced=1, verify updated values.
    - Test: `applyImport` transaction rollback — if an entry has invalid periodRefId (no matching period in import), the entire transaction fails and DB is unchanged.
    - Test: `createBackup` writes a file to the expected directory (use a temp dir override or mock path_provider).
    - Test: `_pruneBackups` keeps only 3 files when 5 exist.
    - Test: progress callback fires correct counts.
