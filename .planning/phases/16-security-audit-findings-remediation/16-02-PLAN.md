<task type="auto">
  <name>Task 1: Add lockout state persistence and minimum PIN enforcement to LockService</name>
  <files>apps/ptrack/lib/features/lock/lock_service.dart, apps/ptrack/lib/features/lock/pin_setup_sheet.dart, apps/ptrack/test/features/lock/lock_service_test.dart</files>
  <action>
  **LockService changes** (`lock_service.dart`):

  1. Add secure storage keys for lockout state:
     - `lock_failed_attempts` (int as string)
     - `lock_lockout_until` (ISO 8601 timestamp string)

  2. Add a `DateTime Function()` clock parameter (default `DateTime.now`) for testability.

  3. Add methods:
     - `Future<void> recordFailedAttempt()` — increment failed count in secure storage; if count reaches threshold, calculate and store lockout-until timestamp. Thresholds: 3 attempts → 30s, 5 → 60s, 7 → 300s (5min), 10+ → 900s (15min). Use exponential-ish backoff.
     - `Future<bool> isLockedOut()` — read lockout-until from storage, compare to clock. Return true if still locked.
     - `Future<Duration> lockoutRemaining()` — return remaining Duration (or Duration.zero).
     - `Future<void> resetLockoutState()` — clear failed attempts and lockout-until. Call on successful PIN verify.

  4. In `verifyPin()`: after successful verification, call `resetLockoutState()`. Do NOT add lockout check here — that goes in the view model.

  5. In `deletePinData()`: also clear lockout keys so reset doesn't leave stale lockout state.

  **PIN minimum** (`pin_setup_sheet.dart`):

  6. Change `minLength: 4` to `minLength: 6` in the `PinEntryWidget` within `_buildCreatePinStep`. This enforces 6-digit minimum on new PINs.

  7. Update the create hint text (via existing l10n key `pinSetupCreateHint`) — the executor should check if the hint mentions "4 digits" and update the EN/DE ARB values to say "at least 6 digits" if so. If the hint is generic ("Enter a PIN"), leave it.

  **Tests** (`lock_service_test.dart`):

  8. Create (or extend) `apps/ptrack/test/features/lock/lock_service_test.dart`:
     - Test `recordFailedAttempt` increments counter in storage.
     - Test `isLockedOut` returns true when lockout-until is in the future.
     - Test `isLockedOut` returns false when lockout-until is in the past.
     - Test `resetLockoutState` clears counter and lockout.
     - Test `verifyPin` success calls `resetLockoutState`.
     - Use a mock `FlutterSecureStorage` (or in-memory map) and injectable clock.
  </action>

  </verify>
  <done>LockService persists failed attempts + lockout-until in FlutterSecureStorage; lockout durations escalate with attempts; successful PIN verify resets state; minimum PIN length is 6; all tests pass.</done>

<task type="auto">
  <name>Task 2: Wire lockout into LockViewModel and LockScreen UI</name>
  <files>apps/ptrack/lib/features/lock/lock_view_model.dart, apps/ptrack/lib/features/lock/lock_screen.dart, apps/ptrack/test/features/lock/lock_view_model_test.dart</files>
  <action>
  **LockViewModel changes** (`lock_view_model.dart`):

  1. Add observable state:
     - `bool isLockedOut = false`
     - `int lockoutSecondsRemaining = 0`

  2. Add method `Future<void> checkLockoutState()` that reads `lockService.isLockedOut()` and `lockService.lockoutRemaining()`, updates the two fields above, and calls `notifyListeners()`.

  3. In `verifyPin()`:
     - Before calling `lockService.verifyPin(pin)`, check `await lockService.isLockedOut()`. If locked out, set `isLockedOut = true`, update `lockoutSecondsRemaining`, call `notifyListeners()`, and return early (do not attempt verification).
     - On failed PIN: call `await lockService.recordFailedAttempt()`, then `await checkLockoutState()`.
     - On success: lockout is already reset inside `lockService.verifyPin` (from Task 1).

  4. Add a `Timer?` that ticks every second while `isLockedOut` is true, decrementing `lockoutSecondsRemaining` and calling `notifyListeners()`. When countdown reaches 0, set `isLockedOut = false` and cancel the timer.

  5. Dispose the timer in `dispose()`.

  **LockScreen changes** (`lock_screen.dart`):

  6. When `vm.isLockedOut` is true:
     - Disable the PIN entry widget (grey out or replace with a message).
     - Show a countdown: "Try again in X seconds" using `vm.lockoutSecondsRemaining`.
     - Use existing l10n pattern — add `lockScreenLockout` key to EN/DE ARBs: e.g. "Too many attempts. Try again in {seconds} seconds." / "Zu viele Versuche. Erneut in {seconds} Sekunden."

  7. When `vm.isLockedOut` is false, show the normal PIN entry.

  **Tests** (`lock_view_model_test.dart`):

  8. Create (or extend) `apps/ptrack/test/features/lock/lock_view_model_test.dart`:
     - Test that `verifyPin` with locked-out service does not call `lockService.verifyPin`.
     - Test that failed attempts transition `isLockedOut` to true after threshold.
     - Test that countdown decrements and eventually clears lockout.
     - Use a mock `LockService` (mocktail).
  </action>

    <automated>cd apps/ptrack && fvm flutter test test/features/lock/ --reporter expanded</automated>
  </verify>
  <done>LockViewModel checks lockout before verify, starts countdown timer on lockout, clears on expiry; LockScreen shows lockout message with countdown; widget/unit tests cover lockout transitions and timer behavior.</done>
</task>
