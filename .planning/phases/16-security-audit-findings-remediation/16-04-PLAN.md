  <action>
  **Password policy** (`export_wizard_screen.dart`):

  1. Define `static const int minPasswordLength = 8;` in the wizard screen (or a shared constants file).

  2. In `_onPasswordNext()`:
     - After checking `a != b` (mismatch), add a check: if `a.length < minPasswordLength`, show validation error "Password must be at least 8 characters" (add EN/DE ARB keys: `exportPasswordTooShort`). Return without proceeding.
     - The "Skip" path remains unchanged — it sets password to null (unencrypted export). This is intentional: users can still export unencrypted, but if they choose encryption, the password must be strong.

  3. Add real-time validation feedback: set the `TextField` `errorText` when the user types and the length is below minimum. Clear it when length is sufficient.

  **ExportViewModel** (`export_view_model.dart`):

  4. In `setPassword()` (or wherever password is stored on the VM): add a defensive guard that throws `ArgumentError` if password is non-null and shorter than `minPasswordLength`. This prevents programmatic bypass.

  **KDF tuning** (`luma_crypto.dart`):

  5. Update the Argon2id parameters for the EXPORT KDF (not the PIN KDF — that's in `lock_service.dart`):
     - Current: `parallelism: 2, memory: 19456 (19 KB), iterations: 2, hashLength: 32`
     - New: `parallelism: 2, memory: 65536 (64 MB), iterations: 3, hashLength: 32`
     - Rationale: OWASP recommends ≥19 MiB for first-line defense and ≥46 MiB for sensitive data. 64 MiB targets ~0.5–1s on modern mobile (acceptable UX for a one-time export). Document this in a comment above the `_kdf` field.

  6. Add a doc comment on the `_kdf` field:
     ```dart
     /// Argon2id parameters: 64 MiB memory, 3 iterations, parallelism 2.
     /// Targets ~0.5–1s on mid-range mobile (2024+). Tuned per OWASP
     /// recommendations for sensitive-data protection. Benchmark on target
     /// devices if changing.
     ```

  **Backward compatibility note:** Existing encrypted exports used the old KDF params. The `decrypt` path must remain compatible because salt + params are re-derived from the password alone — Argon2id params are fixed in code, not stored in the file. This means old exports encrypted with the weaker params will NOT decrypt with the new params.

  **CRITICAL: Handle backward compatibility.**
  7. The current format does NOT store KDF params in the file. To maintain backward compatibility:
     - Option A: Bump `lumaFormatVersion` to 2 for new exports. On import, if `format_version == 1`, use old KDF params; if `format_version == 2`, use new params.
     - Option B: Store KDF params in the encrypted blob header.

     Use **Option A** — it's simpler and aligns with the existing version check infrastructure:
     - In `export_schema.dart`, change `const int lumaFormatVersion = 1;` to `const int lumaFormatVersion = 2;` and add `const int lumaMinSupportedFormatVersion = 1;`.
     - In `luma_crypto.dart`, add a second `_kdfLegacy` field with the OLD parameters. Add a `formatVersion` parameter to `decrypt` that selects which KDF to use.
     - In `import_service.dart`, pass `meta.formatVersion` to `LumaCrypto.decrypt`.
     - In `import_service.dart` `parseFileMeta`, update the version check: accept `formatVersion >= lumaMinSupportedFormatVersion && formatVersion <= lumaFormatVersion`.

  <files>packages/ptrack_data/lib/src/export/luma_crypto.dart, packages/ptrack_data/lib/src/export/export_service.dart, packages/ptrack_data/lib/src/export/import_service.dart, packages/ptrack_data/test/export/luma_crypto_test.dart, packages/ptrack_data/test/export/export_import_roundtrip_test.dart</files>
  <action>
  **LumaCrypto AAD** (`luma_crypto.dart`):

  1. Add optional `List<int>? aad` parameter to both `encrypt` and `decrypt`:
     ```dart
     static Future<Uint8List> encrypt(
       List<int> plaintext,
       String password, {
       List<int>? aad,
     }) async { ... }
     ```

  2. Pass `aad` to `_aes.encrypt(plaintext, secretKey: secretKey, nonce: nonce, aad: aad ?? const [])` and similarly for `_aes.decrypt(box, secretKey: secretKey, aad: aad ?? const [])`.

  3. When `aad` is provided, the GCM MAC authenticates both the ciphertext AND the AAD. If the AAD differs on decrypt, the MAC check fails → `SecretBoxAuthenticationError`.

  **Export side** (`export_service.dart`):

  4. In `_finish()`, when encrypting:
     - Serialize `exportData.meta.toJson()` to canonical JSON bytes: `final metaBytes = utf8.encode(jsonEncode(exportData.meta.toJson()));`
     - Pass `aad: metaBytes` to `LumaCrypto.encrypt`.
     - The outer JSON envelope still contains `"meta": { ... }` in plaintext — but now the MAC covers it.

  **Import side** (`import_service.dart`):

  5. In `parseFileData()`, in the encrypted branch:
     - Before decrypting, serialize the meta from the outer envelope to the same canonical form: `final metaBytes = utf8.encode(jsonEncode(meta.toJson()));`
     - Pass `aad: metaBytes` to `LumaCrypto.decrypt(encryptedBytes, password, aad: metaBytes)`.
     - If the metadata was tampered, GCM auth fails → existing `SecretBoxAuthenticationError` catch maps to `LumaDecryptionException('Incorrect password or corrupted file')`.

  6. **Backward compatibility**: format v1 exports were encrypted without AAD. The `aad` parameter must only be passed for format v2+:
     ```dart
     final aadBytes = meta.formatVersion >= 2
         ? utf8.encode(jsonEncode(meta.toJson()))
         : null;
     decrypted = await LumaCrypto.decrypt(encryptedBytes, password, aad: aadBytes);
     ```

  **Tests** (`luma_crypto_test.dart` + `export_import_roundtrip_test.dart`):

  7. Create `packages/ptrack_data/test/export/luma_crypto_test.dart`:

     - `decrypt with wrong AAD fails (SecretBoxAuthenticationError)`
     - `decrypt without AAD on AAD-encrypted blob fails`
     - `encrypt/decrypt without AAD (legacy) still works`

     - Legacy test: create a format v1 encrypted export (using old KDF, no AAD) → import succeeds with correct password.
