  <name>Task 1: Threat model and release checklist</name>
  <files>docs/THREAT_MODEL.md, docs/RELEASE_CHECKLIST.md</files>
  <action>
  **Threat model** (`docs/THREAT_MODEL.md`):

  1. Create `docs/THREAT_MODEL.md` covering local attacker classes as recommended by the security audit:

     Structure:
     - **Scope**: Local-first period tracker with no network requirements. All data on-device.
     - **Assets**: Health data (periods, symptoms, notes), user preferences, PIN credentials, encryption keys.
     - **Attacker classes** (from audit recommendation):
       a. **Casual access** (borrowed/stolen unlocked device): Mitigated by app lock (PIN + biometrics + lockout).
       b. **Rooted device / filesystem access**: Mitigated by SQLCipher DB encryption, encrypted backups, key material in platform secure storage (Keystore/Keychain).
       c. **Forensic extraction** (full device image): Partially mitigated — encrypted DB and backups resist offline analysis; temp file zero-fill reduces residual data; platform secure storage provides hardware-backed key protection where available.
     - **Residual risks**: Unlocked device with app already open; compromised platform secure storage on rooted device; clipboard sniffing during share.
     - **Trust boundaries**: App process ↔ filesystem, app ↔ share sheet, app ↔ platform secure storage.

  2. Keep it concise (under 100 lines). This is a reference, not a full security assessment.

  **Release checklist** (`docs/RELEASE_CHECKLIST.md`):

  3. Create `docs/RELEASE_CHECKLIST.md` with pre-release security checks:

     Structure:
     - **Signing**: Release APK uses dedicated keystore (not debug), CI verification step passes.
     - **Encryption defaults**: DB encrypted via SQLCipher, auto-backups encrypted by default, export format version is current.
     - **PIN/Lock**: Minimum 6-digit PIN enforced, lockout active after 3 failures, lockout persists across restart.
     - **Import/Export**: Max import file size enforced, password minimum enforced for encrypted exports, share warning dialog active.
     - **Reset/Wipe**: Factory reset removes DB, backups, preferences, PIN, encryption keys. Failures are logged.
     - **CI checks**: `flutter analyze` clean, all tests pass, ARB parity check, release signing verification.

  4. Format as a markdown checklist (`- [ ]`) for manual use during release prep.
  </action>

  </verify>
  <done>THREAT_MODEL.md covers casual/rooted/forensic attacker classes with mitigations; RELEASE_CHECKLIST.md lists all security-sensitive pre-release checks; both files are concise and actionable.</done>
