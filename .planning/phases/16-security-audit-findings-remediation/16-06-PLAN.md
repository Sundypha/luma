  **Luma export delivery** (`luma_export_delivery_io.dart`):

  1. Change the temp file name from `result.filename` (predictable: `luma-backup-2026-04-10.luma`) to a randomized name:
     ```dart
     import 'dart:math';

     String _randomHex(int length) {
       final rng = Random.secure();
       return List.generate(length, (_) => rng.nextInt(256).toRadixString(16).padLeft(2, '0')).join();
     }
     ```
     Use: `final file = File('${dir.path}/luma-${_randomHex(8)}.luma');`
     This produces names like `luma-a3f7b21c.luma` — non-guessable.

  2. Strengthen the cleanup in the `finally` block:
     - Current: `file.deleteSync()` wrapped in try/catch with silent `catch (_)`
     - New: add a small delay (`await Future.delayed(Duration(milliseconds: 100))`) before delete to give the share sheet time to copy the file, then delete. Log failure via `debugPrint` instead of silent catch.
     - Also attempt to zero-fill the file before deletion for defense-in-depth (write zeros of the same length, then delete). This is best-effort — don't fail if it doesn't work:
       ```dart
       try {
         final length = file.lengthSync();
         file.writeAsBytesSync(Uint8List(length));
         file.deleteSync();
       } catch (e) {
         debugPrint('Temp file cleanup failed: $e');
       }
       ```

  4. If the PDF share path creates temp files, apply the same zero-fill + delete cleanup.
