<task type="auto">
  <name>Task 2: Create cross-platform version bump script</name>
  <files>tool/bump_version.dart</files>
  <action>
Create `tool/bump_version.dart` — a standalone Dart script (no extra dependencies beyond `dart:io` and `dart:core`) that automates the version bump workflow.

**Usage:** `dart run tool/bump_version.dart <major|minor|patch> [--dry-run] [--tag]`

**Behavior:**

1. **Parse arguments:** Accept positional `major`, `minor`, or `patch`. Optional flags: `--dry-run` (print changes without writing), `--tag` (auto-commit and create annotated tag after writing).

2. **Read current version** from `apps/ptrack/pubspec.yaml`:
   - Find the line matching `version: X.Y.Z+N`
   - Parse into major, minor, patch (int) and buildNumber (int)

3. **Compute new version** based on bump level:
   - `major` → (major+1).0.0
   - `minor` → major.(minor+1).0
   - `patch` → major.minor.(patch+1)
   - New build number = old build number + 1 (always increments; versionCode must be monotonically increasing for Android)

4. **Update pubspec.yaml** — replace the `version:` line in-place. Do NOT rewrite or reformat the rest of the file.

5. **Prepend CHANGELOG.md section** — insert after the `## [Unreleased]` line:
   ```
   
   ## [X.Y.Z] - YYYY-MM-DD
   
   ### Added
   
   ### Changed
   
   ### Fixed
   
   ```
   Where YYYY-MM-DD is today's date. The developer fills in the subsections before committing.

6. **Print summary:**
   ```
   Bumped: 1.0.0+1 → 1.0.1+2
   
   Updated:
     - apps/ptrack/pubspec.yaml
     - CHANGELOG.md
   
   Next steps:
     1. Edit CHANGELOG.md — fill in Added/Changed/Fixed sections
     2. git add apps/ptrack/pubspec.yaml CHANGELOG.md
     3. git commit -m "release: v1.0.1"
     4. git tag -a v1.0.1 -m "Release 1.0.1"
     5. git push origin main --follow-tags
   ```

7. **If `--tag` flag** is provided (and NOT `--dry-run`):
   - Run `git add apps/ptrack/pubspec.yaml CHANGELOG.md`
   - Run `git commit -m "release: vX.Y.Z"`
   - Run `git tag -a vX.Y.Z -m "Release X.Y.Z"`
   - Print: "Committed and tagged vX.Y.Z. Push with: git push origin main --follow-tags"

8. **If `--dry-run`** — print what WOULD change without writing any files or running git commands.

**Error handling:**
- Exit 1 with message if pubspec.yaml not found or version line missing
- Exit 1 if CHANGELOG.md not found (must exist; Task 1 creates it)
- Exit 1 if `## [Unreleased]` marker not found in CHANGELOG.md
- Exit 1 if bump level argument is missing or invalid

**Cross-platform:** Use `dart:io` `Process.runSync` for git commands (only when `--tag`). File I/O via `File` class. No platform-specific shell syntax.
