<task type="auto">
  <name>Task 1: PDF document builder with all content sections</name>
  <files>apps/ptrack/lib/features/pdf_export/pdf_document_builder.dart</files>
  <action>
    Create `PdfDocumentBuilder` class using the `pdf` package (`package:pdf/pdf.dart` + `package:pdf/widgets.dart`).

    **API:**
    ```dart
    class PdfDocumentBuilder {
      Future<Uint8List> build({
        required PdfReportData data,
        required PdfSectionConfig config,
        required PdfContentStrings strings,
      });
    }
    ```

    `PdfContentStrings` is a simple holder class (defined in same file or adjacent) populated from `AppLocalizations` at call site — avoids passing BuildContext into the builder. Fields: `disclaimer`, `reportTitle`, `overviewHeading`, `cycleHistoryHeading`, `cycleChartHeading`, `daySummaryHeading`, `notesHeading`, `generatedOn`, `dateRange`, `totalCycles`, `avgCycleLength`, `avgPeriodDuration`, `shortestCycle`, `longestCycle`, `flowLabel`, `painLabel`, `moodLabel`, `notesLabel`, `dateLabel`, `daysLabel`, `nDays` (plural-aware format string), `noDataForRange`, `cycleStartLabel`, `cycleLengthLabel`. Include labels for each FlowIntensity/PainScore/Mood enum value.

    **Page setup:**
    - `PdfPageFormat.a4` as default (CONTEXT.md says follow locale; A4 is safe default, user can override via print dialog later).
    - Page margins: 2cm all sides.
    - Header font: Helvetica Bold 16pt; body: Helvetica 10pt; table header: Helvetica Bold 9pt; table body: Helvetica 9pt.

    **Document structure (top to bottom, only enabled sections render):**

    1. **Header (always):** "Luma — Period Report" (localized via `strings.reportTitle`), generation date formatted per locale, date range formatted per locale. Minimal — no large logos. Small app-name line.

    2. **Disclaimer (always):** Short fixed text: "This report is an informational export generated from self-reported data. It is not a diagnostic tool. Accuracy depends on the completeness and consistency of user entries. Consult a qualified healthcare professional for medical decisions." (Localized.) Rendered in italic 8pt with a thin top border to visually separate.

    3. **Overview stats section** (if `PdfSection.overviewStats` enabled): Render `CycleStatsSummary` fields as a compact key-value block:
       - Total completed cycles: N
       - Average cycle length: N.N days
       - Average period duration: N.N days
       - Shortest / Longest cycle: N / N days
       - Flow distribution: compact inline (Light: N, Medium: N, Heavy: N)
       - Pain distribution: compact inline
       - Mood distribution: compact inline
       If stats are all null/zero, show a "No data in selected range" note.

    4. **Cycle history section** (if `PdfSection.cycleHistory` enabled): Table with columns: Cycle Start Date | Cycle Length (days). One row per `CycleLengthEntry`. Use `pw.Table` with alternating light-gray row backgrounds for readability.

    5. **Cycle chart section** (if `PdfSection.cycleChart` enabled AND `data.cycleLengths.length >= 2`): Simple bar chart or line chart of cycle lengths over time using `pw.Chart` + `pw.BarDataSet` from the pdf package. X-axis: cycle start dates (abbreviated). Y-axis: days. Keep it simple — no fancy styling. If fewer than 2 data points, skip this section silently (not enough for a meaningful chart).

    6. **Day summary table section** (if `PdfSection.daySummaryTable` enabled): Table with columns: Date | Flow | Pain | Mood. One row per `DaySummaryRow`. Use localized enum labels (e.g. FlowIntensity.light → "Light" / "Leicht"). Alternating row backgrounds. If list is empty, show "No day-level data in selected range".

    7. **Notes log section** (if `PdfSection.notesLog` enabled): For each `NoteEntry`, render date as a bold header line followed by the notes text. Separated by small vertical spacing. If list is empty, show "No notes in selected range".

    8. **Footer (always):** Small centered text at page bottom: "Generated by Luma · {date}" — page numbers via `pw.Footer`.

    **Edge cases:**
    - When `config.hasAnySections == false`: render only header + disclaimer + "This report contains metadata only. Enable data sections for cycle details." + footer.
    - When data is sparse for a section, show a "no data" message rather than an empty table.

    Use `pw.MultiPage` for automatic page breaking. All text must come from `PdfContentStrings` — no hard-coded English in the builder.
  </action>
