  <action>
    **pdf_export_view_model.dart:** Create `PdfExportViewModel extends ChangeNotifier`:
    - Fields: `PdfSectionConfig _config`, `bool _generating`, `Uint8List? _pdfBytes`, `String? _error`.
    - Constructor: takes `PeriodRepository` and `PeriodCalendarContext`. On init, load saved config from `PdfSectionConfig.loadSaved()`.
    - Getters: `config`, `isGenerating`, `pdfBytes`, `error`, `hasPreview`.
    - `void updatePreset(PdfExportPreset preset)` — replaces enabled sections from preset, keeps date range, notifies. Saves to persistence.
    - `void toggleSection(PdfSection section)` — adds/removes from enabled set, notifies. Saves.
    - `void updateDateRange(DateTime start, DateTime end)` — updates range, notifies. Does NOT re-generate automatically (user triggers generate).
    - `Future<void> generate(String locale)`:
      1. Set `_generating = true`, `_error = null`, notify.
      2. Snapshot current periods from repository via `watchPeriodsWithDays().first` (or a one-shot query).
      3. Call `PdfDataCollector().collect(...)` with snapshot, config, calendar, locale.
      4. Call `PdfDocumentBuilder().build(...)` with report data, config, and `PdfContentStrings` (passed in or resolved here).
      5. Set `_pdfBytes`, `_generating = false`, notify.
      6. On error: set `_error` with user-friendly message, `_generating = false`, notify.
    - `void clearPreview()` — nulls `_pdfBytes`, notifies.
    - `String generateFilename()` — `luma-report-YYYY-MM-DD.pdf` using current date.

    **pdf_export_screen.dart:** `PdfExportScreen` StatefulWidget:
    - Takes `PeriodRepository` and `PeriodCalendarContext` params.
    - Creates `PdfExportViewModel` in `initState`, disposes in `dispose`.
    - Uses `ListenableBuilder` to rebuild on VM changes.
    - Layout (Scaffold with AppBar "Export PDF Report"):
      1. **Preset selector:** Row of 3 `ChoiceChip` widgets (Summary, Standard, Full). Selecting one calls `vm.updatePreset()`. Highlight chip matching current section set (if it matches a preset exactly; otherwise none highlighted — user is in custom mode).
      2. **Advanced section toggles:** `ExpansionTile` labeled "Sections" that expands to show a `SwitchListTile` for each `PdfSection`. Each shows the section name and an icon. Toggling calls `vm.toggleSection()`.
      3. **Date range:** Two `ListTile` rows showing "From: {date}" and "To: {date}". Tapping either opens `showDatePicker()` (restricted to valid range). Default: 12 months ago → today.
      4. **Generate button:** `FilledButton.icon` (icon: `Icons.picture_as_pdf`) labeled "Generate Preview". Disabled while generating. Calls `vm.generate(Localizations.localeOf(context).languageCode)`. Shows `CircularProgressIndicator` while generating.
      5. **Error display:** If `vm.error` is non-null, show a `MaterialBanner` or `SnackBar` with error text.
    - On successful generation (`vm.pdfBytes != null`), push `PdfPreviewScreen`.

    **pdf_preview_screen.dart:** `PdfPreviewScreen` StatelessWidget:
    - Takes `Uint8List pdfBytes` and `String filename`.
    - Scaffold with AppBar "Preview" and a share action button (`Icons.share`).
    - Body: Use `printing` package's `PdfPreview` widget with `build: (_) async => pdfBytes` for scrollable preview. Set `canChangePageFormat: false`, `canDebug: false`.
    - Share action: Write bytes to temp file (same pattern as `luma_export_delivery_io.dart`), invoke `SharePlus.instance.share(ShareParams(files: [XFile(path)]))`. Clean up temp file after share completes.
    - On Linux (where PdfPreview may not work), fall back to: save via `FilePicker.saveFile(fileName: filename, bytes: pdfBytes)`.
    - Handle errors with a `SnackBar`.
