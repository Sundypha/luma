**`diary_view_model.dart`** — paginated loading with search, tag, and date range filters:

```dart
import 'dart:async';
import 'package:flutter/foundation.dart';
import 'package:ptrack_data/ptrack_data.dart';

const _kPageSize = 30;

class DiaryViewModel extends ChangeNotifier {
  DiaryViewModel(this.diaryRepository) {
    _loadNextPage();
  }

  final DiaryRepository diaryRepository;

  // Pagination state
  final List<StoredDiaryEntry> _loadedEntries = [];
  bool _hasMore = true;
  bool _isLoadingMore = false;
  int get _currentOffset => _loadedEntries.length;

  // Filter state
  String _searchQuery = '';
  Set<int> _activeTagIds = {};
  DateTimeRange? _dateFilter;  // null = no date restriction

  // Derived output
  List<StoredDiaryEntry> _filteredEntries = [];

  List<StoredDiaryEntry> get filteredEntries => _filteredEntries;
  String get searchQuery => _searchQuery;
  Set<int> get activeTagIds => _activeTagIds;
  DateTimeRange? get dateFilter => _dateFilter;
  bool get hasMore => _hasMore;
  bool get isLoadingMore => _isLoadingMore;

  /// Called by DiaryScreen's ScrollController when near the end of the list.
  Future<void> loadNextPage() async {
    if (_isLoadingMore || !_hasMore) return;
    _isLoadingMore = true;
    notifyListeners();
    final page = await diaryRepository.getEntriesPage(
      offset: _currentOffset,
      limit: _kPageSize,
    );
    _loadedEntries.addAll(page);
    _hasMore = page.length == _kPageSize;
    _isLoadingMore = false;
    _rebuildFiltered();
  }

  Future<void> _loadNextPage() => loadNextPage();

  /// Resets pagination and reloads from offset 0 (used after save/delete).
  Future<void> reload() async {
    _loadedEntries.clear();
    _hasMore = true;
    await loadNextPage();
  }

  void updateSearch(String query) {
    _searchQuery = query;
    _rebuildFiltered();
  }

  void toggleTag(int tagId) {
    if (_activeTagIds.contains(tagId)) {
      _activeTagIds = {..._activeTagIds}..remove(tagId);
    } else {
      _activeTagIds = {..._activeTagIds, tagId};
    }
    _rebuildFiltered();
  }

  void setDateFilter(DateTimeRange? range) {
    _dateFilter = range;
    _rebuildFiltered();
  }

  void clearFilters() {
    _searchQuery = '';
    _activeTagIds = {};
    _dateFilter = null;
    _rebuildFiltered();
  }

  bool get hasActiveFilters =>
      _searchQuery.isNotEmpty || _activeTagIds.isNotEmpty || _dateFilter != null;

  void _rebuildFiltered() {
    var result = List<StoredDiaryEntry>.from(_loadedEntries);
    if (_searchQuery.isNotEmpty) {
      final lower = _searchQuery.toLowerCase();
      result = result
          .where((e) => e.data.notes?.toLowerCase().contains(lower) == true)
          .toList();
    }
    if (_activeTagIds.isNotEmpty) {
      result = result
          .where((e) => e.tags.any((t) => _activeTagIds.contains(t.id)))
          .toList();
    }
    if (_dateFilter != null) {
      final start = _dateFilter!.start;
      final end = _dateFilter!.end;
      result = result.where((e) {
        final d = e.data.dateUtc;
        return !d.isBefore(start) && !d.isAfter(end);
      }).toList();
    }
    _filteredEntries = result; // already desc from repository
    notifyListeners();
  }

  @override
  void dispose() {
    super.dispose();
  }
}
```

Note: `diaryRepository.getEntriesPage(offset, limit)` is added to `DiaryRepository` in Plan 18-02 (see that plan's Task 2 action). It returns `Future<List<StoredDiaryEntry>>` ordered by `dateUtc` descending.

**`diary_screen.dart`:**

Structure:
- `Scaffold` with:
  - `appBar: AppBar(title: Text(l10n.navDiary), actions: [_filterIconButton(context)])` — filter icon opens DateRangePicker (see below).
- Body: `Column` with:
  1. **Search bar**: `Padding` wrapping `TextField` with search icon. On changed: `viewModel.updateSearch(value)`.
  2. **Tag filter row** (shown only when `watchTags` has tags): `StreamBuilder<List<DiaryTag>>` on `viewModel.diaryRepository.watchTags()` → horizontally scrollable `SingleChildScrollView` of `Wrap` with `FilterChip` widgets. Active when `viewModel.activeTagIds.contains(tag.id)`.
  3. **Active date range badge** (shown when `viewModel.dateFilter != null`): A `Chip` showing the formatted range with an `×` to clear it. Tap `×` → `viewModel.setDateFilter(null)`.
  4. **Entry list** (fills remaining space): `ListenableBuilder(listenable: viewModel, builder: ...)` wrapping `ListView.builder` with a `ScrollController`:
     - `itemCount: viewModel.filteredEntries.length + (viewModel.hasMore ? 1 : 0)`
     - Last item when `hasMore`: a centered `CircularProgressIndicator` (pagination sentinel).
     - `ScrollController` listener: when position > `maxScrollExtent - 200`, call `viewModel.loadNextPage()`.
     - If `viewModel.filteredEntries.isEmpty` and `viewModel.hasActiveFilters`: "No entries match your filters." center text.
     - If `viewModel.filteredEntries.isEmpty` and no filters: empty state with `Icon(Icons.book_outlined)` + "No diary entries yet. Tap + to add your first entry."
     - Otherwise: each item is a `_DiaryEntryCard`.
- **FAB**: `floatingActionButton: FloatingActionButton(onPressed: _openNewEntry, child: Icon(Icons.add), tooltip: l10n.diaryFormTitleNew)`

IconButton(
  icon: Icon(
    Icons.filter_list,
    color: viewModel.dateFilter != null
        ? Theme.of(context).colorScheme.primary  // highlighted when active
        : null,
  ),
  tooltip: l10n.diaryFilterIconTooltip,
  onPressed: () async {
    final picked = await showDateRangePicker(
      context: context,
      firstDate: DateTime(2000),
      lastDate: DateTime.now(),
      initialDateRange: viewModel.dateFilter,
    );
    if (picked != null) viewModel.setDateFilter(picked);
  },
),
