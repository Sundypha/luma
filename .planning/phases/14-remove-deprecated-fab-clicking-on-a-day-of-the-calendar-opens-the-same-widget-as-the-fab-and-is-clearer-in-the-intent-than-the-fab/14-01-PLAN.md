Remove the deprecated global FloatingActionButton from `TabShell` so period logging is driven by clearer, contextual entry points: the Home **Today** card (for “today”) and the **calendar day** bottom sheet (for any day). Purpose: Align shell UX with the mental model “pick a day, then act” and reduce visual clutter. `DayDetailSheet` and `HomeScreen._handleTodayQuickAction` already implement the same repository flows the FAB used (`markDay` / `showSymptomFormSheet`).

Output: No FAB; updated localization (drop unused tooltips); `logging_test.dart` updated to target the surviving affordances.
</objective>

## Current FAB behavior (to retire)

`TabShell._onFabPressed`: if today not marked → `_homeVm.markToday()`; else → `showSymptomFormSheet` for today.

## Replacement surfaces

- **Home:** `TodayCard` → `onTodayAction` → `HomeScreen._handleTodayQuickAction` (mark today if needed, then symptom sheet).
- **Calendar:** `TableCalendar.onDaySelected` → `showDayDetailSheet` → `DayDetailSheet` actions (`dayDetailHadPeriod`, `dayDetailAddSymptoms`, etc.).

Executor must not reintroduce a second global FAB or a duplicate floating action for the same flows.

## Localization

Remove `fabTooltipMarkToday` and `fabTooltipAddSymptoms` from `app_en.arb` / `app_de.arb`, then regenerate localizations (`flutter gen-l10n` from `apps/ptrack`). Run `dart run tool/arb_de_key_parity.dart` from repo root.

## Tests

All `find.byType(FloatingActionButton)` usages in `logging_test.dart` must be replaced:

- Home-tab scenarios: tap `TodayCard` primary buttons using stable text from ARB (`todayMarkPeriodCta`, `todayAddSymptomsCta`, or `AppLocalizations` in test setup).
- Calendar-tab scenario: switch to Calendar (`navCalendar` / label from l10n), tap the **today** day cell in `TableCalendar`, then tap the sheet CTA that opens symptoms (e.g. `dayDetailAddSymptoms` text) when today is already marked — mirror the old test’s intent.

Assert `find.byType(FloatingActionButton)` finds **nothing** on `TabShell` after pump (optional one-line guard test).
