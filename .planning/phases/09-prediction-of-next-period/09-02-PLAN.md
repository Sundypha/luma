**ensemble_coordinator.dart** — Create `EnsembleCoordinator`:

```
class EnsembleCoordinator {
  EnsembleCoordinator({List<PredictionAlgorithm>? algorithms})
    : _algorithms = algorithms ?? _defaultAlgorithms;

  static final List<PredictionAlgorithm> _defaultAlgorithms = [
    MedianBaselineAlgorithm(),
    EwmaAlgorithm(),
    BayesianAlgorithm(),
    LinearTrendAlgorithm(),
  ];
```

Key methods:

- `EnsemblePredictionResult predictNext({required List<StoredPeriod> storedPeriods, required PeriodCalendarContext calendar})`:
  1. Extract `PredictionCycleInput` list using existing `predictionCycleInputsFromStored()`.
  2. Compute shared period duration: median of observed bleeding durations across completed periods. If no duration data, default to 5 days.
  3. Run each algorithm's `predict(cycles)`. Collect non-null `AlgorithmPrediction` outputs. Pass shared duration to each algorithm (via the `defaultDuration` param from Plan 01).
  4. For each algorithm output, generate a set of predicted calendar days: `{predictedStartUtc, predictedStartUtc+1, ..., predictedStartUtc+duration-1}` (all UTC midnight).
  5. Build `dayConfidenceMap`: for each unique predicted day across all algorithms, count how many algorithms include it. `Map<DateTime, int>`.
  6. Detect milestone: compare `activeAlgorithmCount` against a previously stored count (accept `int? previousActiveCount` parameter). Generate milestone message per CONTEXT.md:
     - 2→3 active (or 1→2): "With {cycles} cycles logged, your prediction now uses {N} methods for better accuracy."
     - 3 active (EWMA + Bayesian + Median all active): "3 cycles logged — all core methods are now active."
     - 4 active (LinearTrend joined): "With 5 cycles, trend detection is now active."
  7. Build `consensusPrediction` (backward-compat `PredictionResult`): derive from the algorithm with most cycles considered (MedianBaseline), or synthesize from ensemble midpoint. Use existing `PredictionCoordinator` result as the consensus since it's the most established algorithm.
  8. Build merged explanation steps + formatted explanation text using new `formatEnsembleExplanation()`.
  9. Return `EnsemblePredictionResult`.

- `Future<EnsemblePredictionResult> predictNextFromRepository({required PeriodRepository repository, required PeriodCalendarContext calendar, int? previousActiveCount})`: Async wrapper that loads stored periods first.
