    ---
    **lock_gate.dart** — Lifecycle-aware gate widget that overlays `LockScreen` when locked.

    ```dart
    class LockGate extends StatefulWidget {
      const LockGate({
        super.key,
        required this.lockService,
        required this.child,
        required this.onReset,    // VoidCallback — passed through to LockScreen → ForgotPinSheet
        this.onLockNow,           // Optional callback for 'Lock now'; set by parent
      });

      final LockService lockService;
      final Widget child;
      final VoidCallback onReset;
      final VoidCallback? onLockNow;
    ```

    Implementation (`_LockGateState`):
    - `bool _isLocked` — initialized in `initState` to `widget.lockService.isEnabled` (cold start: if lock is enabled, start locked).
    - `late final AppLifecycleListener _listener` — created in `initState`:
      - `onPause`: if `lockService.isEnabled` → `setState(() => _isLocked = true)`
      - `onResume`: `setState(() {})` — triggers rebuild; _isLocked is already true from onPause
    - Dispose: `_listener.dispose()`
    - `build`: if `_isLocked` → return `LockScreen(lockService: lockService, onUnlocked: () => setState(() => _isLocked = false), onForgotPin: () => showForgotPinSheet(context, onReset: widget.onReset))`; else → return `widget.child`.
    - Expose a `void lockNow()` method (called externally for "Lock now" button): `setState(() => _isLocked = true)`.
    - For `onLockNow` wiring: `LockSettingsScreen` receives `onLockNow` as a VoidCallback. Since `LockGate` creates its child lazily, the simplest wiring is to use a `GlobalKey<_LockGateState>` on `LockGate` and call `key.currentState?.lockNow()` from `LockSettingsScreen.onLockNow`. Alternatively pass a `ValueNotifier<bool> lockSignal` from `LumaApp` that `LockGate` listens to. Prefer the `GlobalKey` approach for simplicity — it matches the project's no-Riverpod patterns.

    - Implement `_resetApp()`:
      ```dart
      Future<void> _resetApp() async {
        // 1. Delete lock data
        await widget.lockService?.deletePinData();
        await widget.lockService?.disableLock();
        // 2. Clear SharedPreferences (onboarding state, lock flags)
        final prefs = await SharedPreferences.getInstance();
        await prefs.clear();
        // 3. Close and delete the SQLite database file
        await widget.database?.close();
        try {
          final dir = await getApplicationDocumentsDirectory();
          final dbFile = File(path.join(dir.path, 'ptrack.db'));
          if (await dbFile.exists()) await dbFile.delete();
        } catch (_) {}
        // 4. Route to onboarding
        if (mounted) setState(() => _screen = AppScreen.onboarding);
      }
      ```
      Add `import 'dart:io'` and `import 'package:path/path.dart' as path` (or use `dart:io` join — check if `path` package is in pubspec; if not, use `'${dir.path}/ptrack.db'`). `path_provider` is already in pubspec for `getApplicationDocumentsDirectory`.

    ```dart
    class LockSettingsTile extends StatelessWidget {
      const LockSettingsTile({super.key, required this.lockService});
      final LockService lockService;

      @override

        return ListTile(
          leading: const Icon(Icons.lock_outlined),

              builder: (_) => LockSettingsScreen(

                  // Wire via an onReset callback on LockSettingsTile OR use Navigator.pushReplacement
                  // after reset. Since this is app-layer, simplest: add onReset to LockSettingsTile constructor.
                },
                onLockNow: () {
                  // Wire via GlobalKey on LockGate — see note below
                },
              ),
            ));
          },
        );
      }
    }
    ```

    **Wiring `onReset` and `onLockNow` through `LockSettingsTile`:**

    The cleanest approach without Riverpod: add `VoidCallback onReset` and `VoidCallback onLockNow` to `LockSettingsTile`. Pass them from `TabShell`, which receives them from `LumaApp` via constructor. Update `TabShell` constructor:
    ```dart
    const TabShell({
      ...
      required this.lockService,
      required this.onReset,      // forwarded to LockSettingsTile → LockSettingsScreen → ForgotPinSheet
      required this.onLockNow,    // forwarded to LockSettingsTile → LockSettingsScreen
    });
    ```

    In `LumaApp.build`, the `LockGate` parent holds `_lockGateKey`. Pass `onLockNow: () => _lockGateKey.currentState?.lockNow()` to `TabShell`. Declare `_lockGateKey = GlobalKey<_LockGateState>()` in `_LumaAppState` — but `_LockGateState` is private. Work around: make `LockGate` expose a `lock()` method via a public key type by defining `LockGateState` as a public class:
    ```dart
    class LockGateController {
      _LockGateState? _state;
      void lock() => _state?.setState(() => _state!._isLocked = true);
    }
    ```
    Pass `LockGateController` from `LumaApp` down to `LockGate` and `TabShell`. `LockGate` registers itself: `widget.controller?._state = this` in `initState`. `TabShell` calls `controller.lock()` from `onLockNow`.

    Alternative simpler approach (preferred): Use a `ValueNotifier<bool> _lockNowSignal` in `_LumaAppState`. `LockGate` listens to it (addListener in initState). When the notifier fires true, `LockGate` sets `_isLocked = true`. `TabShell.onLockNow` calls `lockNowSignal.value = true`. This avoids public state classes.

    Choose whichever approach compiles cleanly — both are acceptable. Document the chosen approach in the summary.
