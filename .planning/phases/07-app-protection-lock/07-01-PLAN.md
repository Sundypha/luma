  <action>
    Create `apps/ptrack/lib/features/lock/lock_service.dart`.

    `LockService` is app-layer only (no database dependency). It depends on:
    - `flutter_secure_storage` for PIN hash + salt (import `package:flutter_secure_storage/flutter_secure_storage.dart`)
    - `local_auth` for biometric prompts (import `package:local_auth/local_auth.dart`)
    - `cryptography` + `cryptography_flutter` for Argon2id (already in workspace; `import 'package:cryptography/cryptography.dart'`)
    - `shared_preferences` for the isEnabled/isBiometricsEnabled flags (import `package:shared_preferences/shared_preferences.dart`)
    - `dart:convert` for `base64`
    - `dart:math` for `Random.secure()`

    Constructor accepts all deps as optional named params (with defaults) for testability:
    ```dart
    LockService({
      FlutterSecureStorage? storage,
      LocalAuthentication? localAuth,
      SharedPreferences? prefs,
    })
    ```

    Storage keys (private constants):
    - `_hashKey = 'lock_pin_hash'`
    - `_saltKey = 'lock_pin_salt'`
    - `_enabledKey = 'lock_enabled'`
    - `_biometricsKey = 'lock_biometrics_enabled'`

    State getters (synchronous, reads from injected SharedPreferences):
    - `bool get isEnabled => _prefs?.getBool(_enabledKey) ?? false`
    - `bool get isBiometricsEnabled => _prefs?.getBool(_biometricsKey) ?? false`

    Methods to implement:
    - `Future<bool> canUseBiometrics()` — returns `await _auth.canCheckBiometrics || await _auth.isDeviceSupported()`. Catch any exception and return false.
    - `Future<void> createPin(String pin)` — generate 16 random bytes as salt, hash with Argon2id(parallelism: 1, memory: 8192, iterations: 3, hashLength: 32), write base64(salt) to `_saltKey` and base64(hashBytes) to `_hashKey` in secure storage.
    - `Future<bool> verifyPin(String pin)` — read stored salt and hash; return false if either is null. Re-derive hash from input+salt with same Argon2id params; compare byte-by-byte to stored hash.
    - `Future<bool> authenticateWithBiometrics(String localizedReason)` — call `_auth.authenticate(localizedReason: localizedReason, biometricOnly: true)`. Catch `LocalAuthException` and return false. Never throw.
    - `Future<void> enableLock()` — `await _prefs?.setBool(_enabledKey, true)`.
    - `Future<void> disableLock()` — `await _prefs?.setBool(_enabledKey, false)`, `await _storage.delete(key: _hashKey)`, `await _storage.delete(key: _saltKey)`.
    - `Future<void> setBiometricsEnabled(bool enabled)` — `await _prefs?.setBool(_biometricsKey, enabled)`.
    - `Future<void> deletePinData()` — delete `_hashKey` and `_saltKey` from secure storage (used by destructive reset).
    - `Future<bool> hasPin()` — returns true if `_hashKey` read returns non-null.

    Use `compute()` or the cryptography_flutter binding for Argon2id. Since `ptrack_data` already depends on `cryptography_flutter`, the native binding is available on the platform. Call `deriveKeyFromPassword` normally — the Flutter binding dispatches off the main isolate on mobile.

    Note on Argon2id params: use lighter params than export (parallelism=1, memory=8192 kB, iterations=3) to keep unlock under ~150ms while remaining brute-force resistant for a ≤6-digit PIN with device-side rate limiting.

    Create `apps/ptrack/test/features/lock/lock_service_test.dart`.

    Use mocktail. Mock `FlutterSecureStorage` and `LocalAuthentication`. For `SharedPreferences`, use `SharedPreferencesWithCache` via `InMemorySharedPreferencesAsync` pattern (same as Phase 3 onboarding tests) — or simpler: pass a real `SharedPreferences` instance via `SharedPreferences.setMockInitialValues({})` in setUp.

    Test cases:
    - `isEnabled` is false by default (no SharedPreferences entry)
    - `createPin` writes hash and salt to secure storage (two `write` calls, different keys)
    - `verifyPin` returns true for the same PIN used in `createPin` (integration-style: real Argon2id, real SharedPreferences mock, mock storage that stores values in a Map)
    - `verifyPin` returns false for a wrong PIN
    - `verifyPin` returns false if no hash stored (returns null from storage)
    - `enableLock` sets `lock_enabled = true` in SharedPreferences
    - `disableLock` sets `lock_enabled = false` and deletes hash/salt from storage
    - `authenticateWithBiometrics` returns true when `_auth.authenticate` returns true
    - `authenticateWithBiometrics` returns false when `LocalAuthException` is thrown
    - `canUseBiometrics` returns true when `canCheckBiometrics` is true
    - `deletePinData` calls delete on both `_hashKey` and `_saltKey`

    For the `verifyPin` integration test: use a fake `FlutterSecureStorage` that backs writes/reads with an in-memory `Map<String, String>` (implement via mocktail `when` returns).

    Mock `LocalAuthentication.authenticate` using mocktail. Note: `local_auth` v3 uses `biometricOnly` as a named param directly on `authenticate()` (not via `AuthenticationOptions` — that class was removed in v3.0.0).
