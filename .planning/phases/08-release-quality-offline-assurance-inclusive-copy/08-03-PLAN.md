  <action>
Run automated checks to confirm zero network dependencies in the production build. These are read-only verification commands — no file modifications.

**Check 1: Android release manifest — no INTERNET permission**

```bash
rg "INTERNET" apps/ptrack/android/app/src/main/AndroidManifest.xml
```

Expected: zero matches. The INTERNET permission should only appear in `debug/` and `profile/` manifests (for Flutter DevTools), not in `main/` (release).

**Check 2: No HTTP/network imports in production Dart code**

```bash
rg -i "import.*http|HttpClient|Uri\.https?\b|package:dio|package:http\b" apps/ptrack/lib/ packages/ptrack_domain/lib/ packages/ptrack_data/lib/
```

Expected: zero matches. No production code should import any HTTP client.

**Check 3: No network-dependent packages in pubspec**

Review `apps/ptrack/pubspec.yaml` and `packages/*/pubspec.yaml` for any dependency that requires network at runtime. The known dependency list (from research):
- `flutter`, `drift`, `shared_preferences`, `flutter_secure_storage`, `local_auth`, `table_calendar`, `smooth_page_indicator`, `timezone`, `share_plus`, `file_picker`, `cryptography`, `cryptography_flutter`, `cupertino_icons` — all local-only ✓.

Flag any NEW dependency added since research was conducted.

**Check 4: No Firebase, analytics, or telemetry SDKs**

```bash
rg -i "firebase|analytics|crashlytics|sentry|amplitude|mixpanel|telemetry" apps/ptrack/pubspec.yaml packages/*/pubspec.yaml
```

Expected: zero matches.

**Check 5: iOS — no App Transport Security exceptions needed**

```bash
rg "NSAppTransportSecurity" apps/ptrack/ios/Runner/Info.plist
```

Expected: zero matches (or standard Flutter boilerplate only — no custom exceptions for external hosts).

Document all results. If ANY check fails, investigate and report — do not silently skip.
